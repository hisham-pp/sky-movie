# Security Policy

## Supported Versions

Security updates are provided for the current default branch and the latest
published release, when releases are available.

## Reporting a Vulnerability

Please do not open a public issue for a suspected security vulnerability.
Instead, contact the maintainers through the project's preferred private
security contact channel.

Include as much detail as you safely can:

- A description of the vulnerability.
- Steps to reproduce it.
- Affected versions, platforms, or configurations.
- Any known impact.
- Suggested fixes or mitigations, if you have them.

## Response Expectations

Maintainers will try to acknowledge valid reports promptly, investigate the
issue, and coordinate a fix before public disclosure. Timelines may vary based
on maintainer availability and the severity of the report.

## Scope

This policy covers vulnerabilities in the Sky Movie source code and official
project packages. Third-party services, dependencies, and deployment
environments may have their own security policies.

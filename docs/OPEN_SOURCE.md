# Open Source Guide

Sky Movie is open source so that people can inspect, use, improve, and learn
from the project.

## Project Documents

- `LICENSE` explains the terms for using and redistributing the project.
- `CONTRIBUTING.md` explains how to contribute.
- `CODE_OF_CONDUCT.md` sets expectations for respectful participation.
- `SECURITY.md` explains how to report vulnerabilities.
- `SUPPORT.md` explains where to get help.
- `GOVERNANCE.md` explains how project decisions are made.

## Contributor Path

1. Read the contribution guide.
2. Look for an existing issue or open a short proposal.
3. Make a focused change.
4. Test the change locally.
5. Open a pull request with a concise explanation.

## Maintainer Path

1. Triage issues and pull requests regularly.
2. Keep discussions respectful and on topic.
3. Ask for tests or docs when they reduce future confusion.
4. Merge changes that improve the project without unnecessary complexity.
5. Keep releases, security notes, and setup instructions current.

## License Note

The project currently uses the MIT License. If the license ever changes,
update `LICENSE`, package metadata, and this document together.

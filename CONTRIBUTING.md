# Contributing

Thanks for your interest in improving Sky Movie. This project welcomes fixes,
documentation improvements, UI polish, bug reports, and thoughtful feature
ideas.

## Getting Started

1. Fork the repository and create a branch from the default branch.
2. Install dependencies with `pnpm install`.
3. Make your changes in a focused branch.
4. Run the relevant checks before opening a pull request.
5. Open a pull request with a clear summary and screenshots when the change
   affects the UI.

## Development

This repository is a PNPM workspace. Use the scripts in `package.json` and the
package-level scripts in `website` or `desktop-app` for local development.

Before submitting a change:

- Keep the change scoped to one problem or feature.
- Update docs when behavior or setup steps change.
- Add or update tests when the change affects behavior.
- Avoid committing generated files unless the project explicitly tracks them.

## Pull Requests

Good pull requests are easy to review. Please include:

- What changed and why.
- How you tested the change.
- Screenshots or recordings for visual changes.
- Any follow-up work that remains.

Maintainers may ask for changes before merging. That review loop is part of
keeping the project healthy and useful for everyone.

## Reporting Bugs

When reporting a bug, include:

- What you expected to happen.
- What actually happened.
- Steps to reproduce the issue.
- Your operating system, browser, and app version when relevant.
- Logs, screenshots, or console output if they help explain the problem.

## Suggesting Features

Feature requests are welcome. Please describe the user problem first, then the
solution you have in mind. This makes it easier to compare ideas and choose an
approach that fits the project.
